# 인터넷은 뭐고, 웹은 뭐였을까

처음에는 `웹 = 인터넷`처럼 느껴졌지만, 공부를 하다 보니 둘은 역할이 꽤 다르다는 걸 알게 되었다. 이 글은 그 구분을 나 스스로 설명할 수 있을 정도로 정리해 두기 위한 기록이다.

## 인터넷: 전 세계를 잇는 통신 기반

- 케이블, 라우터, 스위치 같은 **물리 장비** 위에  
- IP(Internet Protocol)가 얹히고  
- 그 위에서 TCP, UDP 등 여러 프로토콜이 돌아간다.

즉 인터넷은 **“패킷을 여기서 저기로 전달하는 거대한 네트워크 인프라”**에 가깝다.

## 웹: 인터넷 위에 얹힌 정보 공유 시스템

웹은 인터넷 위에서 **HTTP(HyperText Transfer Protocol)**, **URL(Uniform Resource Locator)**, **HTML(HyperText Markup Language)**을 이용해 문서와 리소스를 주고받는 시스템이다.

- 브라우저는 URL로 서버에 HTTP 요청을 보내고  
- 서버는 HTML, CSS, JS, 이미지 등을 응답으로 돌려준다.

그래서 웹을 이렇게 말할 수 있다.

> 웹은 인터넷 위에서 돌아가는, 링크로 연결된 정보 공유 시스템이다.

## 이 구분이 개발자에게 왜 중요한가

- 웹만 알고 있으면 “브라우저 안에서만” 생각하게 된다.  
- 인터넷 레벨을 이해하면, 게임 서버, 채팅 서버, 파일 전송, IoT 등 **브라우저 밖의 세계**를 떠올릴 수 있다.

이 글은 “웹 개발자”로 머물지 않고, **인터넷 위에서 돌아가는 다양한 시스템**을 상상할 수 있는 개발자가
