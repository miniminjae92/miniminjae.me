# 바퀴는 다시 만들지 않되, 구조는 이해하고 싶었다

“내 블로그를 완전 처음부터 만들까, 아니면 템플릿을 쓸까?”  
이 질문에서 꽤 오래 머물렀다. 결론은 이거였다.

> 바퀴는 다시 만들지 말고,  
> 대신 **바퀴가 어떻게 돌아가는지는** 이해하자.

## 내가 선택한 스택

- Next.js + TypeScript
- Tailwind CSS
- MDX(or Velite 기반 콘텐츠 시스템)
- Vercel 배포

즉, “지금 웹 생태계에서 잘 굴러가는 바퀴들”을 모아서 쓴 셈이다.

## 그럼에도 뜯어본 부분들

템플릿을 그대로 쓰면서도, 특히 이런 부분은 꼭 확인했다.

- `app` 디렉터리 구조
- `layout.tsx`에서 폰트, 테마, 메타데이터 설정
- `src/content` 구조와 슬러그 생성 방식
- `mdx` → 페이지로 매핑되는 흐름

## 나에게 남은 기준 한 줄

> 남이 만든 바퀴를 쓰되,  
> **축, 방향, 속도는 내가 설계한다.**

이 기준이 있으면, 라이브러리에 의존하면서도 주도권을 잃지 않을 수 있다.
