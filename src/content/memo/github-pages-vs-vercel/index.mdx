# GitHub Pages vs Vercel 간단 비교

블로그나 포트폴리오를 어디에 올릴지 고민할 때 참고하기 위한 메모.

## GitHub Pages

**장점**

- 완전 무료, GitHub 연동이 자연스럽다.
- 정적 사이트(Jekyll, Hugo, 빌드된 Next.js 등)에 적합하다.

**단점**

- 빌드 파이프라인을 직접 세팅해야 하는 경우가 많다.
- 동적 기능, 서버리스 함수 같은 건 별도 서비스가 필요하다.

## Vercel

**장점**

- Next.js 1등 시민, 기본 설정만으로도 바로 배포 가능
- PR마다 프리뷰 URL이 자동으로 생성된다.
- 서버리스 함수, Edge Functions 등 동적 기능 지원

**단점**

- 플랫폼 의존성이 생긴다.
- 무료 플랜의 한계를 고려해야 한다.

정리하면,

> 완전 정적 블로그만이면 GitHub Pages도 충분.  
> Next.js의 기능을 적극 활용하고 싶다면 Vercel이 더 편하다.
