오늘 블로그 구조를 Insight, Memo, Log로 다시 정리했다.  
이름 하나 바꾸는 일이 왜 이렇게 오래 걸렸는지 생각해보니,  
카테고리는 결국 **내가 개발을 어떻게 바라보는지** 그대로 드러나는 부분이었기 때문이다.

- Insight는 개념을 나만의 언어로 재구성하는 곳
- Memo는 바로 꺼내 쓸 실무 지식을 적재하는 곳
- Log는 시간이 지나야만 만들어지는 기록

특히 Log는 그동안 애매하게 흩어져 있었다.  
무언가를 배울 때, 세팅을 뜯어볼 때, 하루의 시행착오를 되짚을 때  
이건 어느 카테고리에 넣어야 하지? 하는 고민이 항상 따라붙었다.

이제는 그럴 필요 없다.  
회고는 Log에, 정제된 개념은 Insight에, 빠른 참고는 Memo에 쌓이면 된다.

사소하지만, 이런 정리는 앞으로의 흐름을 결정한다.  
정리되지 않은 상태에서는 배움도 관리되지 않는다.  
카테고리를 바꾼 건 단순한 UI 변경이 아니라  
**내 학습 방식의 프로토콜을 다시 잡은 것**에 가깝다.

오늘의 작은 전환점.
